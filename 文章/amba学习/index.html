Remarks
Protocol The Protocol layer is the top-most layer in the bus protocol architecture. The function of the Protocol layer is to:
• Generate and process requests and responses at the protocol nodes.
• Define the permitted cache state transitions at the protocol nodes that include caches.
• Define the transaction flows for each request type. • Manage the protocol level flow control.
Network The function of the Network layer is to:
• Packetize the protocol message.
• Determine, and add to the packet, the source and target node IDs required to route the packet over the interconnect to the required destination.
Link The function of the Link layer is to:
• Provide flow control between network devices.
• Manage link channels to provide deadlock free switching across the network.

2 APB

2.1 APB概况

APB提供低功耗接口，常用在低带宽和不需要高性能总线的外部设备上。APB时非流水线结构，所有信号仅与时钟**上升沿**有关，每个传输至少消耗**两个**时钟周期。

2.2 写操作

分为两种写操作：**无**等待状态写传输与**有**等待状态写传输。
信号 来源 描述
PCLK 时钟源 时钟
PADDR APB Bridge 地址总线，最多32位
PWRITE APB Bridge 方向，高电平为写，低电平为读
PSEL APB Bridge 选择信号
PENABLE APB Bridge 使能信号
PWDATA APB Bridge 写入的数据，最高32位
PREADY Slave interface 准备好，从器件利用该信号拓展APB传输

2.2.1 无等待状态写传输

两个周期：

  1. 第一个周期T1（称为建立周期），PADDR、PWRITE、PSEL、PWDATA在PCLK上升沿寄存。

  2. 第二个周期T2，使能PENABLE和PREADY。

     PENABLE表示传输访问周期的开始，PREADY表示在PCLK的下一个上升沿从设备完成传输。

  3. PADDR、PWDATA和控制信号一直保持有效，直到T3周期完成传输。

                    ![image.png](https://cdn.nlark.com/yuque/0/2020/png/1571366/1594387330128-99cc5f47-2a8a-4ec7-813d-57ec6f868a8b.png#align=left&display=inline&height=164&margin=%5Bobject%20Object%5D&name=image.png&originHeight=295&originWidth=533&size=86917&status=done&style=none&width=297)

2.2.2 有等待状态写传输

这是一种方式拓展传输的方式，通过PREADY信号拓展传输。在访问周期，当PENABLE高时，拉低PREADY。当PENABLE为低时，PREADY可以为任何值。 注意：在传输结束时可以不立即更改地址和写信号，保持当前状态直到下一个传输，这样可以降低功耗。

2.3 读操作

读传输和写传输一样也可以分为两种

2.3.1 无等待状态读传输

  在这里无等待状态**读**传输的和无等待状态**写**传输的区别在于PWRITE（这个信号看之前的表格在读操作时应该拉低），以及PRDATA在读传输操作完成前，一直要提供数据。

2.3.2 有等待状态读传输

 和有等待写传输一样，都是在PENABLE为高时拉低PREADY。

2.4 错误响应

信号 来源 描述
PSLVERR Slave interface 传输失败 
   通过**PSLVERR**来指示APB传输错误条件。
   只有在传输最后一个周期PENABLE和PREADY、PSEL都为高，PSLVERR才是有效的。

3 AHB

3.1 AHB概况

AHB用于高性能、高时钟频率的系统结构，典型的如ARM与RAM、Flash、Bridge之间的连接。
AHB具有以下**特性**：

（1）burst transfers【促发传输】 （2） single clock edge operation【单时钟沿操作】 （3）non-tristate implementation（非三态的） （4）wider data bus configurations【更宽的数据总线配置】 典型 AHB 总线系统﻿

3.2 AHB总线结构

  AHB总线基于一个中心多路复用器互联机制。
  AHB 总线系统有 Master、 Slave 和 Infrastructure 构成。 Infrastructure 由 Arbiter，数据多路，地址控制多路，译码器构成。
  总线上所有主设备都可以驱动地址和控制信号，用于传输。Arbiter【仲裁器】用于决定连接到所有从设备的主设备地址和控制信号。

3.3 AHB操作

3.3.1 信号介绍

信号 描述
HTRANS 指出当前的传输状态
NONSEQ、SEQ、 IDLE、 BUSY
HSIZE 指出当前传输的大小
HBURST burst类型
HRESP 从设备发给主设备的传输状态
OKAY、ERROR、、
HREADY
HTRANS状态 含义
NONSEQ
  • 表明一次单个数据传输
  • 一次burst传输的第一个数据
  • 地址和控制信号与上一次传输无关 | | SEQ |
  • 表明burst接下来传输的数据
  • 地址和上一次传输的地址是相关的 | | IDLE | | | BUSY | 无效 |

流水线传送：先地址再数据。

开始传输前，授权总线主设备访问总线。通过主设备对连接到仲裁器请求信号的确认。然后，指示授权主设备使用总线。 通过驱动地址、控制信号，一个授权的总线设备启动AHB传输。（提供地址、方向、传输宽带等信息）。 AHB有两种猝发模式：（1）增量猝发，在地址边界回卷（2）回卷猝发。在一个特殊的边界回卷。

每个传输的构成如下： (1一个地址和控制周期 （2）一个或多个数据周期。

3.3.2一次无需等待状态的简单传输

  1. HCLK的第一个周期，主设备驱动总线上的地址和控制信号。
  2. 在第二个上升沿，从设备采样地址和控制信号。
  3. 当从设备采样地址和控制后，驱动正确的响应信号。在第三个上升沿，总线主设备采样这个信号。
